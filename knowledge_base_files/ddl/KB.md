# Ddl Section

The DDL (Data Definition Language) section contains comprehensive documentation on the database table schemas used in the analytics database. This includes detailed column definitions and semantic descriptions that are crucial for analysts and developers to understand the data structure and relationships. This section is particularly useful for those involved in data modeling, ETL processes, and report generation.

## Contents

- **dwh_d_games**: Schema and column definitions for the games dimension table, detailing game-specific attributes.
- **dwh_d_players**: Schema and column definitions for the players dimension table, including player-specific information.
- **dwh_d_player_nicknames**: Schema and column definitions for the player nicknames dimension table, capturing alternative player names.
- **dwh_d_teams**: Schema and column definitions for the teams dimension table, providing team-specific details.
- **dwh_f_player_awards**: Schema and column definitions for the player awards fact table, documenting player achievements and recognitions.
- **dwh_f_player_boxscore**: Schema and column definitions for the player boxscore fact table, containing detailed player performance metrics.
- **dwh_f_player_team_seasons**: Schema and column definitions for the player team seasons fact table, tracking player participation across seasons.
- **dwh_f_player_tracking**: Schema and column definitions for the player tracking fact table, offering insights into player movements and actions.
- **dwh_f_team_boxscore**: Schema and column definitions for the team boxscore fact table, summarizing team performance statistics.
- **dwh_f_team_championships**: Schema and column definitions for the team championships fact table, recording team championship achievements.
